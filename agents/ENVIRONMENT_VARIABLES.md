# Environment Variables — Colvin Content OS

Complete reference for all environment variables. All secrets stored in `.env.local` locally and as Vercel/GitHub Secrets in CI. Never in code, never in markdown.

---

## Storage Rules

| Location | Used For | Key Type |
|----------|---------|---------|
| `.env.local` | Local development | ALL secrets |
| Vercel Environment Variables | Production + Preview deployments | ALL secrets |
| GitHub Secrets | CI/CD (GitHub Actions) | ALL secrets |
| Code, markdown, logs | NEVER | NEVER |

---

## Required Variables

These must be set before any workflow runs.

### Supabase

```
NEXT_PUBLIC_SUPABASE_URL=https://iuzlbtfevzkerehmluqj.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<service_role_key>
```

> **CRITICAL:** `SUPABASE_SERVICE_ROLE_KEY` must NEVER be in a `NEXT_PUBLIC_*` variable. Server-side only.

> `NEXT_PUBLIC_SUPABASE_URL` is safe to expose — it is the project URL, not a secret.

### OpenAI

```
OPENAI_API_KEY=sk-...
```

Used for: GPT-4o (complex reasoning, content generation), GPT-4o-mini (routing, scoring, dedup)

### Telegram

```
TELEGRAM_BOT_TOKEN=<bot_token>
TELEGRAM_CHAT_ID=<alfred_chat_id>
```

Used for: Daily brief delivery, review queue alerts, P1/P2 incident notifications

### Firecrawl

```
FIRECRAWL_API_KEY=<key>
```

Used for: All web research via Firecrawl MCP

### Resend (Email)

```
RESEND_API_KEY=re_...
```

Used for: All outbound email via Gabriel. Drafts only — Alfred approves before sending.

**Setup steps:**
1. Create free account at [resend.com](https://resend.com) (3,000 emails/month free)
2. Go to API Keys → Create API Key
3. Add to `.env.local` as `RESEND_API_KEY=re_...`
4. Add to GitHub Secrets for CI runs
5. (Optional but recommended) Verify your domain in Resend dashboard for each lane sender address

**Until domain is verified:** all emails send from `onboarding@resend.dev` (Resend test address — functional but shows test sender)

### Katrina Gate Notifications

```
KATRINA_TELEGRAM_CHAT_ID=<katrina_chat_id>
```

Used for: Separate Telegram notification to Katrina when high-risk content (first_keys_indy, funding_ready_indiana, girls_got_game) is queued for review. Alfred still sees it too — this ensures Katrina gets her own notification.

**Setup:** Katrina needs to message `@colvinenterprises_bot` (or your bot) first. Get her chat ID from the bot's `/getUpdates` endpoint.

## Active MCP Variables

### Remotion MCP

```
REMOTION_MCP_URL=http://localhost:3001
```

> Default is localhost:3001. Update to your actual Remotion MCP server URL.
> Required for: Daily Remotion Content Workflow, render triggers, composition listing

### Gemini (Google AI)

```
GOOGLE_API_KEY=<key>
```

Used for: Gemini MCP — second-opinion checks, cross-model verification

## Optional Variables

These enhance the system but are not required for core workflows.

```
# Playwright MCP
PLAYWRIGHT_MCP_URL=http://localhost:3002

# Next.js
NEXTAUTH_SECRET=<random_string>
NEXTAUTH_URL=http://localhost:3000

# Vercel (auto-set by Vercel — do not set manually)
VERCEL_URL=
VERCEL_ENV=

# Analytics (future)
POSTHOG_API_KEY=
```

## Internal Runtime Variables (Auto-Generated)

These are NOT set in `.env.local` — they are generated at runtime by the system.

```
RUN_ID        # Generated by Hermes at workflow start — UUID v4
TRACE_ID      # Generated per stage — UUID v4
```

## Variable Naming Conventions

| Prefix | Meaning |
|--------|---------|
| `NEXT_PUBLIC_` | Safe to expose to browser — use only for non-secret config |
| `SUPABASE_` | Supabase credentials — always server-side |
| `OPENAI_` | OpenAI credentials |
| `TELEGRAM_` | Telegram bot credentials |
| `FIRECRAWL_` | Firecrawl credentials |
| `GOOGLE_` | Google/Gemini credentials |
| `REMOTION_` | Remotion MCP configuration |
| `PLAYWRIGHT_` | Playwright MCP configuration |

## Health Check

To verify all required variables are set:

```bash
npm run health-check
```

Or manually:

```bash
node -e "
const required = [
  'NEXT_PUBLIC_SUPABASE_URL',
  'SUPABASE_SERVICE_ROLE_KEY',
  'OPENAI_API_KEY',
  'TELEGRAM_BOT_TOKEN',
  'TELEGRAM_CHAT_ID',
  'FIRECRAWL_API_KEY',
  'REMOTION_MCP_URL',
];
const missing = required.filter(k => !process.env[k]);
if (missing.length) {
  console.error('Missing env vars:', missing.join(', '));
  process.exit(1);
} else {
  console.log('All required env vars set');
}
"
```
