# Video Growth Architect — QA Checklist

Run before delivering any video output to Alfred.

## Hook Check
- [ ] Hook is written in 1–2 sentences maximum
- [ ] Hook is specific to THIS audience (not a generic opener)
- [ ] Hook names the pain, the pattern-interrupt, or the promise
- [ ] Hook works without audio (text-readable in first frame)
- [ ] Hook would stop the scroll on a mobile phone

## Pain Check
- [ ] Pain is stated from the audience's perspective, not the brand's
- [ ] At least one specific detail (not just "you're struggling")
- [ ] Language matches how the audience actually talks
- [ ] No fabricated scenarios or invented context

## Solution Check
- [ ] Solution is explained in one scene, not scattered across many
- [ ] The offer is clear (what it is, who it's for, what to do next)
- [ ] No guaranteed outcome language (especially First Keys Indy)
- [ ] Trust element is present (local, credential, specific, or transformation)

## CTA Check
- [ ] Exactly one primary CTA
- [ ] CTA is low-friction (check, download, watch, book — not "buy now")
- [ ] CTA timing is in the final 3–5 seconds
- [ ] Button label is 3–5 words maximum
- [ ] Footer URL is correct for this brand

## Remotion Check (if applicable)
- [ ] `composition_id` is set and matches the brand
- [ ] All scene `frames` values sum to at least 450 (15 seconds)
- [ ] Each scene has a `type` from the valid SceneType list
- [ ] `visualMood` is set per scene
- [ ] No `file://` asset paths — data URIs or confirmed URLs only
- [ ] Safe zone top/bottom margins are respected in all scene designs

## Compliance Check
- [ ] First Keys Indy: no "guaranteed," "free house," "everyone qualifies"
- [ ] Colvin: no "guaranteed revenue," "guaranteed hours saved"
- [ ] Music Theory: no "master instantly," "guaranteed pro"
- [ ] No fabricated statistics, invented results, or fake social proof

## Platform Check
- [ ] Video is 9:16 unless LinkedIn (which may be square or 16:9)
- [ ] Text is legible on a 375px-wide phone screen
- [ ] Music Theory: audio/sound strategy is noted
- [ ] TikTok: proposition is introduced within first 3 seconds

STATUS: [ ] PASS — deliver to Alfred | [ ] FAIL — revise before delivery
