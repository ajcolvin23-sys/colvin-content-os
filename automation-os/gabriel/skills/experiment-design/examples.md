# Experiment Design — Examples

## Example 1: Good Experiment Design

**ID:** EXP-001
**Based on:** Evidence Level 3 research — action-oriented CTA labels outperform "Submit"
**Lane:** first_keys_indy

**Hypothesis:**
If we change the form CTA from "Submit" to "Get the Free Buyer Guide," form fill rate will increase by ≥15% because action-oriented labels reduce friction and clarify the benefit.

**Control:** "Submit" button
**Variant:** "Get the Free Buyer Guide" button

**Primary metric:** Form fill rate
**Success threshold:** ≥15% improvement
**Minimum sample:** 200 form views
**Minimum days:** 14

**Risk level:** Low (copy change only, no compliance impact)
**Alfred approval required:** Yes — before launch

## Example 2: Rejected — Two Variables

**Proposed:** Change the hero headline AND the CTA button copy at the same time.

**Why rejected:** Testing two variables simultaneously makes it impossible to know which change drove the result. If conversion goes up, was it the headline or the CTA?

**Correct action:** Design two sequential experiments. Run the headline test first. After conclusion, run the CTA test.

---

## Example 3: Wrong Timing — Early Winner Declared

**What happened:** After 3 days, variant A was showing 40% better results. Gabriel declared it the winner.

**Why it failed:** 3 days of data is statistically unreliable. Day-of-week variation, traffic spikes, and small sample sizes all distort early results.

**Rule added:** Never declare a winner before BOTH minimum sample AND minimum days are met.
