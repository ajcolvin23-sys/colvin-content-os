# Design Pre-Delivery Checklist

- [ ] Page goal defined (one primary visitor action)
- [ ] ICP identified and loaded
- [ ] Mobile-first approach confirmed
- [ ] Section flow is logical (hook → proof → offer → CTA)
- [ ] Two variants proposed with tradeoff explanation
- [ ] One variant recommended with reasoning
- [ ] Reusable components used before new ones created
- [ ] No placeholder images or fake testimonials in spec
- [ ] Design tokens specified
- [ ] Passed to qa-publish-guard before delivery
