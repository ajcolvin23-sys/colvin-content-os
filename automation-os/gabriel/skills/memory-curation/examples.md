# Memory Curation — Examples

## Example 1: Correct Expiry Handling

**Item:** "LinkedIn posts with specific city name in first line get higher Indianapolis engagement"
**Added:** 2026-01-15
**Review date:** 2026-04-15
**Status:** proposed

**Action:** Item is 120 days old with no Alfred review. No experiment was run to validate. Move to `archived-research-memory.md` with note: "Expired without validation. Re-propose if experiment confirms."

---

## Example 2: Contradiction Handling

**Approved item:** "TikTok videos under 30 seconds outperform longer content"
**New finding:** Evidence Level 3 research from TikTok's own creator analytics shows 60–90 second videos with strong storytelling now outperform sub-30-second videos in the educational category.

**Action:** Flag the approved item for Alfred's review. Do not automatically update the approved memory. Write in memory-change-log.md: "Contradiction detected. Approved item on TikTok length may need update. Awaiting Alfred review."

---

## Example 3: What Not to Do

**Situation:** Gabriel promoted a research finding to approved-research-memory.md without Alfred's review because it "seemed obvious."

**Why it failed:** The approved memory is the source of truth Gabriel acts on. Promoting without review bypasses the human-in-the-loop system.

**Rule reinforced:** The only path to approved-research-memory.md is through Alfred's explicit approval.
